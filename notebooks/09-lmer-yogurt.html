library(tidyverse)
library(here)
library(lme4)       # for lmer() — linear mixed-effects models
library(gtsummary)  # for tbl_regression()
library(broom.mixed) # helps gtsummary understand lmer models

The yogurt study has two time points per participant (baseline and after antibiotic), so each person contributes 2 rows of data. We use a mixed model (lmer) to handle this — (1 | pid) means “each person gets their own random intercept.”

# Load and join the data
samples <- read_csv(here("data/group_a_yogurt/00_sample_ids_yogurt.csv"))
qpcr    <- read_csv(here("data/group_a_yogurt/02_qpcr_results_yogurt.csv"))
meta <- read_csv(
  here("data/group_a_yogurt/01_participant_metadata_yogurt.csv"),
  col_types = cols(sex = col_character())
)

# Join qPCR results → sample mapping → participant metadata
df <- samples |>
  left_join(qpcr, by = "sample_id") |>
  left_join(meta, by = c("pid", "arm")) |>
  mutate(
    # Make factors so R knows the reference groups
    time_point = factor(time_point, levels = c("baseline", "after_antibiotic")),
    arm        = factor(arm,        levels = c("unchanged_diet", "yogurt")),
    # Log-transform the bacterial counts (they're huge numbers)
    log_bacteria = log(qpcr_bacteria)
  )

What We Learned

  • lmer() from the lme4 package fits linear mixed-effects models for repeated measures data
  • (1 | pid) adds a random intercept per participant, accounting for the fact that each person has their own baseline level
  • tbl_regression() from gtsummary turns the model output into a publication-ready table with coefficients and confidence intervals
  • add_global_p() adds overall p-values for each term in the model
